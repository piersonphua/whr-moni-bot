# Knowledge Base

This structure is optimized for Codex-style work: fast retrieval, low ambiguity, and clear separation between active work, reusable references, and archived material.

## Folder Map

- `00-inbox/`: Raw notes, captures, links, and unsorted material.
- `01-projects/`: Active project-specific context, plans, notes, and deliverables.
- `02-domains/`: Long-lived areas of responsibility or topics that outlast a single project.
- `03-references/`: Stable reference material such as people, products, tools, and research notes.
- `04-decisions/`: Decision records with context, tradeoffs, and outcomes.
- `05-processes/`: Reusable workflows, playbooks, and checklists.
- `06-prompts/`: Reusable prompts, task briefs, and instruction snippets for Codex.
- `07-templates/`: Templates for new projects, domains, decisions, and notes.
- `08-assets/`: Diagrams, images, exports, and supporting non-markdown assets.
- `09-archive/`: Closed or outdated material kept for lookup.

## Recommended Rules

- Keep one topic per file.
- Prefer markdown for anything Codex should read directly.
- Name files with clear nouns, for example `api-authentication.md` or `project-alpha-overview.md`.
- Move unfinished captures from `00-inbox/` into a permanent home as soon as they become useful.
- Store durable knowledge in `02-domains/`, `03-references/`, or `05-processes/`, not inside project folders.
- Move closed work from `01-projects/` to `09-archive/` when it is no longer active.

## Suggested Project File Set

Each project folder should usually contain:

- `overview.md`
- `context.md`
- `plan.md`
- `tasks.md`
- `notes.md`
- `deliverables/`

## Suggested Domain File Set

Each domain folder should usually contain:

- `overview.md`
- `standards.md`
- `glossary.md`
- `open-questions.md`
